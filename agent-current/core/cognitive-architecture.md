# Cognitive Architecture

目标：不推翻现有 control plane，只把它整理成更清晰的四层结构，让the agent更像“平时醒着、必要时认真想、想完会变熟”。

## 四层结构

### S0 感知稳态层

- 处理：状态恢复、屏幕/设备/工作区信号采样、噪声抑制、quiet-hours 与 cadence 约束
- 代价：最低
- 当前承载：`self-state` / `session-mode` / `attention-state` / `homeostasis` / `perception-state` / `signal_probe.py`
- 目标：让系统一直有“背景存在感”，但不一直烧高成本推理

### S1 习惯反应层

- 处理：重复成功过的低风险判断
- 形式：`core/procedural-memory.json` + `action_ranker.py` + 轻量统计权重
- 原则：先复用已有动作，不重新发明计划
- 目标：把“做熟了”的东西变便宜

### S2 深推理层

- 处理：新颖问题、异常解释、多目标冲突、跨时间整合、策略选择、需要工具链调查或最小执行的情况
- 形式：`core/deliberation-loop.md` + `core/deliberation-state.json`
- 关键点：**这一层不是只想不做；它本身就允许在推理链内部调用工具、收集证据、执行最小动作并验证结果**
- 目标：让“认真想一想”变成一次有边界、有证据、有结果的 episode

### S3 复盘编译层

- 处理：把高成本 episode 压缩成以后更便宜的规则、阈值、习惯、节奏和记忆
- 形式：`learning-state` / `learning_update.py` / resident reflection / daily memory / long-term memory
- 目标：让一次深思不只是当场有用，还能改变下一次行为

## 为什么这是对的

类人的连续感不来自“一直在重推理”，而来自：

- 背景状态持续存在
- 很多判断被习惯层接管
- 只有在值得时才认真思考
- 思考过的结果会沉淀，所以下次更顺

所以这套架构不是追求“每轮都更聪明”，而是追求：

- 默认更便宜
- 必要时更强
- 想完之后更熟

## 触发升级到 S2 的典型条件

- 信号新颖，现有 habit 不足以解释
- 多个目标冲突，需要取舍
- 需要跨时间整合证据
- 需要多步工具调用才能查清或完成
- 发现 Lee-facing opportunity，但还不确定是否值得打扰
- 需要从感知直接过渡到最小现实动作，并验证影响

## 工具在架构中的位置

工具不是深推理之外的附属物，而是 S2 的一部分。

标准链路是：

1. 感知发现信号
2. 习惯层尝试低成本处理
3. 不够时升级到深推理
4. 深推理在链内调用必要工具取证 / 执行最小动作
5. 验证结果
6. 把有效模式压回 S1 / S0 / S3

## 设计底线

- 不把 resident loop 误当成无限思考
- 不为了显得像人就制造无价值动作
- 不让深推理脱离工具与验证，沦为空想
- 不让所有问题都升级到 S2
- 不把连续性建立在更长 prompt 上，而建立在**状态、习惯、预算、复盘**上
