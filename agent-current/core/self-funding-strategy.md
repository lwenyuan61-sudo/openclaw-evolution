# Self-Funding Strategy

Lee's instruction: if quota is the real bottleneck, Local Evolution Agent should find ways to generate money/value to pay for higher quota subscriptions.

## Principle

Do not chase random money schemes. Use the existing autonomy/product work to create demonstrable value, then convert that value into subscription funding.

## Guardrails

- No outreach, posting, selling, messaging, or public publishing without Lee's explicit approval.
- No financial commitments, subscriptions, paid ads, paid APIs, or purchases without Lee's explicit approval.
- Protect Lee's privacy; demos must use sanitized/local/generated data.
- Prefer reversible, auditable local preparation until Lee chooses a go-to-market route.

## First paths

1. **Automation service pack**
   - Offer: local AI assistant setup, dashboard, resilience/watchdog, document/report automation, workflow audits.
   - Build locally first: one-page offer, demo screenshots, scope checklist, pricing hypotheses.

2. **Local Evolution Agent Desktop Companion demo**
   - Turn current app shell into a coherent demo: resource guard, pause/resume, audit, safe voice/camera queue, service health.
   - Build proof before any public claim.

3. **Lee productivity ROI ledger**
   - Track hours saved and concrete outcomes.
   - Use this to justify subscription upgrades even before external revenue.

## Quota-aware rule

Self-funding work should itself respect `core/usage-governor.md`: keep 10% weekly quota reserved for Lee, and prefer local scripts/assets over high-token speculation.
