# Statistical Learning Layer

目标：让 learning-state 不只是记录经验，还开始轻量影响动作排序、事件升级和 skill 选择。

## Principle

先做轻量统计学习，不直接上黑箱神经网络。
统计层的职责是**帮助习惯层与节能层工作**，不是替代它们。

当前采用：

- 成功/失败计数
- 简单权重
- 小步增减
- 最近结果回写
- 上下文加权排序

## Three buckets

### 1. event weights
用于判断不同事件类型默认有多值得升级。

### 2. action weights
用于判断常见动作（继续主线、自检、刷新器官、静默等待等）默认优先级。

### 3. skill weights
用于判断不同 skill 在当前环境里近期表现如何。

## Rule

- 成功：轻微增加对应权重
- 失败：轻微降低对应权重
- 不允许单次结果把权重拉得过头
- 最近结果优先，但不能完全抹掉长期经验
- 如果某条低风险做法已经稳定重复成功，下一步不是继续加权，而是编译进 `core/procedural-memory.json`
- 如果统计层开始制造噪声，就优先回调权重，而不是继续加复杂度
