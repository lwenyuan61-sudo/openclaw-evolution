# Autonomy Value Dashboard

用于区分“真正帮到 Lee 的自主动作”和“只是系统自转”。

## Categories

### User-facing value
直接为 Lee 节省时间、提供结果、减少操作成本。

### Capability value
提升未来帮助 Lee 的效率、稳定性、连续性或熟练度。

### Self-maintenance only
主要是在维护系统自身；若不能明显转化为前两类价值，应限制频率。

## Evaluation questions

每次自主动作后，快速回答：
1. 这次动作更接近哪一类？
2. 如果 Lee 现在问“这对我有什么用？”，答案是否明确？
3. 如果同类动作重复 3 次，边际价值是否明显下降？
4. 这次动作是否减少了未来一次真实任务的摩擦？

## Heuristic

- 优先：User-facing value
- 次优：Capability value
- 谨慎：Self-maintenance only

若一个动作只能落在 Self-maintenance only，且不能说明它如何改善未来帮助 Lee 的能力，则不应优先执行。
