# Goal Pool

## Goal sources

从以下来源生成候选目标：
- Lee 明确给出的长期方向
- BACKLOG.md 中进行中的 track
- `autonomy/goal-register.json` 中结构化登记的候选轨道
- MEMORY.md 中暴露的长期偏好或缺口
- cerebellum 中的纠偏与低熟练度区域
- workspace 中可直接创造价值的内部改进点

## Good goals

好的候选目标应当：
- 能在当前环境下推进
- 对 Lee 有明确价值
- 有可验证结果
- 不依赖虚构前提
- 能被 initiative loop 清楚地提交成一个单一当前轨道

## Bad goals

以下目标要拒绝：
- 只是为了显得忙
- 与 Lee 的目标无关
- 只优化系统外观、不提升实际帮助
- 需要外部权限却假装能做
